# /refactor — Flutter リファクタリング

対象 feature フォルダのコードをリファクタリングします。

## 手順

1. **対象を確認する**
   - 対象の feature フォルダ (`lib/apps/{app}/features/{feature}/`) を指定してください
   - 未指定の場合は現在開いているファイルの feature を対象とします

2. **現状分析**（読み込みのみ・変更なし）
   - 対象フォルダ内の全ファイルを読み込む
   - 問題箇所（長すぎるファイル・重複・命名違反・不要コメント）をリストアップする
   - **ユーザーに確認してから実装に進む**

3. **優先順位で実行**（1ステップ = 1ファイルずつ）
   - Widget 分割（`part` ディレクティブ + `view/widgets/` に private クラス）
   - ViewModel 整理
   - Repository 共通化
   - Model 整理
   - import 整理・命名統一

4. **各ステップ後に確認**
   - 変更差分を提示してユーザーに確認
   - 承認後に次のステップへ

5. **完了確認**
   ```bash
   flutter analyze
   ```
   エラー 0 を確認してから終了。

## 制約（変更不可）
- ロジック・UI 挙動は変えない
- テストを自動追加しない
- 新しいパッケージを導入しない
- ファイル名・クラス名の一括リネームはしない
- 不必要な抽象化レイヤーを追加しない
