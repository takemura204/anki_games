## 1. コンセプト・ポジショニング

### コアコンセプト
> **「ブロックパズルをご褒美に、英単語学習を続けさせる」**

英単語学習がメインの目的。ブロックパズルは学習を継続させるためのゲーミフィケーション機能として機能する。Duolingo のようなストリーク型継続学習をパズルゲームで実現する。

### ターゲットユーザー
- **主**: 英語学習中の日本人（10〜30代）
- **利用シーン**: 通勤・通学など隙間時間での学習習慣化
- **ゴール**: 英単語の定着（スペーシングリピティション）＋ゲームとしての楽しさ

### 他モードとの差別化

| モード | 主目的 | 難易度変動 | 終了条件 |
|--------|--------|-----------|---------|
| クラシック | スコアアタック | なし | 手詰まり |
| クエスト | ノイズ除去 | レベル固定 | ノイズ全消去 |
| タイムアタック | 制限時間内スコア | なし | タイムアップ |
| **英単語×パズル** | **英単語学習継続** | **クイズ成績連動** | **手詰まり** |

## 3. フェーズ詳細

### 3-1. ブロック配置フェーズ

#### UI レイアウト
```
┌─────────────────────────────┐
│  ← | QUIZ MODE | ⚙          │  トップバー
├─────────────────────────────┤
│  SCORE        ×N  BEST      │  ScoreHUD
│  12345      [badge]  9999   │
├─────────────────────────────┤
│                             │
│         8 × 8 ボード         │  BoardWidget
│                             │
├─────────────────────────────┤
│  [ピース1] [ピース2] [ピース3]│  PieceTrayWidget（ドラッグ可）
├─────────────────────────────┤
│           広告バナー          │
└─────────────────────────────┘
```

#### 動作
1. 3ピースをドラッグ&ドロップでボードに配置
2. ライン消去が発生した場合: `quizMultiplier += 1`
3. 3ピース全て配置すると、ピーストレイを空にしてクイズフェーズへ自動遷移
4. スコア計算:
   - 配置スコア: `セル数 × quizMultiplier`
   - ライン消去スコア: `10 × ライン数² × コンボ数 × quizMultiplier`

### 3-2. クイズフェーズ

#### UI レイアウト
```
┌─────────────────────────────┐
│  ← | QUIZ MODE | ⚙          │  トップバー
├─────────────────────────────┤
│  (スコアHUD - 非表示・レイアウト維持)       │
├─────────────────────────────┤
│                             │
│     ┌───────────────┐        │
│     │  ↑ りんご    │        │
│     │              │        │
│  ← │   apple      │ → 走る  │  QuizCard
│  幸せ│              │        │
│     │  ↓ バナナ    │        │
│     └───────────────┘        │
│                             │
├─────────────────────────────┤
│ [□単語1] [□単語2] [□単語3] │  獲得ピースプレビュー（ドラッグ不可）
│  ← 回答済みスロットに表示 →  │  正解=緑枠・不正解=赤枠
├─────────────────────────────┤
│           広告バナー          │
└─────────────────────────────┘
```

#### 動作
1. スワイプカード（`flutter_card_swiper`）で3問連続出題
2. カードをスワイプした方向の選択肢を選択
3. 回答後300ms以内に次のカードへ（誤入力防止）
4. 各問回答のたびにピースをスロットに表示（後述「ピース割り当て」参照）
5. 全3問完了後:
   - **全問正解**: `quizMultiplier += 1`
   - **不正解あり**: `quizMultiplier` 変化なし（リセットしない）
6. ブロック配置フェーズへ自動遷移

#### カード構造
- **中央**: 英単語 または 日本語訳（大きく表示）
- **上下左右チップ**: 4つの選択肢を各方向に配置
  - 正解はランダムな1方向
  - デコイは同カテゴリから優先選出
- **スワイプ中**: 向いている方向のチップをハイライト
- **出題方向**: 英→日 または 日→英（ランダム混在、将来的に設定化予定）

## 4. スコア・倍率設計

### quizMultiplier（クイズ倍率）

| イベント | 変化 |
|---------|-----|
| ライン消去（ブロック配置フェーズ） | `+1` |
| クイズ全問正解（クイズフェーズ終了時） | `+1` |
| クイズ不正解あり | 変化なし（リセットしない） |
| ゲームオーバー | `1` にリセット |

**設計思想**: 英単語を正しく答え続け、かつブロックをうまく消すことで倍率が上昇する。失敗しても一気に崩れない設計で、学習継続意欲を維持する。

### combo（ライン消去コンボ）

| イベント | 変化 |
|---------|-----|
| ライン消去あり | `+1`（消去ライン数分加算） |
| 消去なしのターンが3回連続 | `0` にリセット |

combo は quizMultiplier とは独立して管理される。

### スコア計算式

```
配置スコア = セル数 × quizMultiplier
消去スコア = 10 × ライン数² × comboCount × quizMultiplier
turnScore  = 配置スコア + 消去スコア
```

## 5. ピース割り当てロジック（現行実装）

クイズ回答後、`addQuizPiece(slot, isCorrect)` が呼ばれ、ボードの充填率に応じてピースサイズを決定。

| 条件 | ピース種別 | 目的 |
|------|-----------|-----|
| 充填率 > 70% | 小ピース（≤3セル） | ゲームオーバー防止（充填率に関わらず優先） |
| 充填率 > 50% かつ不正解 | 中ピース（4セル） | 高充填時のペナルティ緩和 |
| それ以外 かつ 正解 | 小ピース（≤3セル） | 正解報酬 |
| それ以外 かつ 不正解 | 大ピース（≥5セル） | 学習促進ペナルティ |

> ⚠️ **今後の変更予定**: ユーザー意向により「正誤でなくピース数で定義したい」という方向に仕様変更を検討。1問＝1ピース（3問＝必ず3ピース）で、ピースの大きさはスコアや倍率等の別要因で決める設計に移行する。

## 6. 学習アルゴリズム

### スペーシングリピティション（重み付き出題）

```
初期 weight = 1.0

正解時:   weight = max(0.1, weight × 0.7)  → 出題頻度↓
不正解時: weight = min(5.0, weight × 1.5)  → 出題頻度↑
```

- 出題確率は `weight` に比例（重み付きランダム選出）
- 苦手な単語ほど多く出題される仕組み
- weight はゲームオーバー後も引き継がれ、累積学習に活用

### 出題データ構造

```dart
// 単語マスタ（CSV由来）
QuizWord { int id, String en, String ja, String category }

// 学習記録（Drift/SQLite）
WordRecord {
  int id,          // QuizWord.id と一致
  int correctCount,
  int incorrectCount,
  double weight,   // 出題重み（初期 1.0）
  DateTime? lastSeenAt,
}
```

## 7. 永続化（セーブ・リストア）

### 保存タイミング
- `placePiece()` 実行後
- `addQuizPiece()` 実行後
- `endQuizPhase()` 実行後
- クイズフェーズ開始時（3ピース全配置検知後）

### 保存データ（SharedPreferences）

| キー | 型 | 内容 |
|------|----|----|
| `block_puzzle_quiz_board` | String | 8×8ボード状態 |
| `block_puzzle_quiz_pieces` | String | 3ピースの状態 |
| `block_puzzle_quiz_score` | int | 現在スコア |
| `block_puzzle_quiz_combo` | int | コンボ数 |
| `block_puzzle_quiz_non_clear_turns` | int | 連続未消去ターン数 |
| `block_puzzle_quiz_multiplier` | int | quizMultiplier |
| `block_puzzle_quiz_is_game_over` | bool | ゲームオーバー状態 |
| `block_puzzle_quiz_is_quiz_phase` | bool | クイズフェーズ中か |

### リストアロジック（`startQuizMode()`）
- 保存データあり かつ `isGameOver=false` → 自動復元
- `isQuizPhase=true` または `pieces` が全 null → クイズフェーズとして復元
- `pieces` に1つ以上ある → ブロック配置フェーズとして復元

### ゲームオーバー後の動作
- `quizMultiplier` はリセット（1に戻す）
- 単語の `weight` は引き継ぎ（学習履歴を維持）
- ゲームオーバーデータのまま保存 → 次回入場時は新規スタート

## 8. UI・UX 詳細

### ScoreHUD 表示
```
[SCORE]         [badge]        [BEST]
12345         ×3 (金バッジ)    9999
```
- `quizMultiplier > 1` のとき: 金色の `×N` バッジを中央表示（アニメーション付き）
- `combo > 1` かつ `quizMultiplier = 1` のとき: アクセントカラーの `xN` コンボバッジ
- 倍率変化時: バッジがポップアニメーション

### フェーズ切り替えアニメーション
- `AnimatedSwitcher`（350ms フェード）でブロック↔クイズレイアウトを切り替え
- ScoreHUD は `Visibility(maintainSize: true)` でクイズ中も高さを維持

### ピーストレイ（クイズフェーズ）
- 正方形フレーム × 3スロット（ドラッグ不可）
- 回答済み: ピース表示 + 正誤色（緑/赤）のフレームと枠線
- 未回答: 薄いグレーフレーム
- スロット下部に単語ラベル（回答後に表示）
