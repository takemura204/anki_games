# ホワイトラベル 収益ロードマップ（月収100万円まで）

資格試験アプリをホワイトラベルで量産し、**24ヶ月で月収100万円**を目指すための数値ベース計画。
実装手順は [add_new_app.md](./add_new_app.md)、戦略の前提は [roadmap.md](./roadmap.md)（資格選定）を参照。

> このドキュメントの収益・DL数値は **すべて仮説**。実測が出たら `KPI` セクションの前提値を更新して再計算する。

## 2. 収益モデル（ARPU 分解）

1DL あたりの生涯価値（LTV）を分解する。ハイブリッドなので「広告 + 買い切り + サブスク」の合算。

```
LTV/DL = 広告LTV + 課金LTV
  広告LTV  = 平均アクティブ月数 × 広告ARPU(月)
  課金LTV  = 課金率 × ( 買い切り比率 × ¥1,500
                      + サブスク比率 × ¥400 × 平均継続月数 )
```

### 基準前提（ベースシナリオ）

| 変数 | 値 | 根拠 |
|---|---|---|
| 課金率（DL→課金） | 2.5% | 資格アプリの一般水準1〜3%、学習意欲高め |
| 買い切り : サブスク | 60% : 40% | 買い切りが取りやすい国内傾向 |
| サブスク平均継続 | 3ヶ月 | 試験までの集中学習期間 |
| 平均アクティブ月数 | 3ヶ月 | 試験前に集中、合格後離脱 |
| 広告ARPU（月/アクティブ） | ¥15 | 演習=インタースティシャル多発・iOS比率高 |
| ストア手数料 | IAP 15%（小規模事業者） | 広告は AdMob 実収ベース |

上記から:
- 課金LTV = 2.5% ×（0.6×1,500 + 0.4×400×3）= 2.5% ×（900 + 480）= **¥34.5**
- 広告LTV = 3 × ¥15 = **¥45**
- **グロス LTV/DL ≈ ¥80**、手数料控除後 **ネット ≈ ¥70/DL**

### 3シナリオ（不確実性の幅）

| シナリオ | 課金率 | 広告ARPU | ネットLTV/DL | 位置づけ |
|---|---|---|---|---|
| 保守 | 1.5% | ¥8 | **¥40** | 競合激戦・ASO伸び悩み |
| ベース | 2.5% | ¥15 | **¥70** | 本計画の標準 |
| 楽観 | 4% | ¥25 | **¥120** | 課金導線・ASO最適化が奏功 |

> **重要**: ネットARPU は運用最適化で伸びる変数。Year1 は ¥45 前後、Year2 に課金導線・広告配置・ASO を磨いて ¥70+ を狙う、という時間軸で計画する。

### 月次収益の算出式

```
月間収益 ≈ Σ(アプリ)  月間新規DL(D) × ネットLTV/DL
```

→ **月収100万円 = Σ(D × ネットARPU) = 1,000,000**
- ネット¥70 なら Σ月間DL ≈ **14,300 DL/月** が必要
- ネット¥45 なら Σ月間DL ≈ **22,200 DL/月** が必要

「アプリ本数 × 1本あたりの月間DL」でこの合計に到達させるのが本ロードマップの骨子。

## 6. KPI と 加速 / 撤退 判断ライン

各アプリはリリース**6ヶ月**で下記を評価する。

| 指標 | 加速ライン（横展開/追加投資） | 撤退・保留ライン |
|---|---|---|
| 主要KWのASO順位 | Top15以内で安定 | 6ヶ月経ってTop30外 |
| 月間新規DL(D) | 成熟見込みDの50%到達 | 成熟見込みの20%未満 |
| 課金率 | 2%以上 | 0.5%未満 |
| ネットARPU/DL | ¥50以上 | ¥25未満 |

- **加速**: 上ラインを満たした資格は「上位級・関連資格」へ即横展開（例: FE→応用情報、FP3→FP2、危険物乙4→他類）。
- **撤退**: 下ラインは新規投資を止め、新年度更新のみ最低メンテ or ストア残置。個人リソースを勝ち筋に集中させる。

## 7. 著作権リスクと安全な進め方（必読）

問題データを「過去問道場等を参照して編集」する方針には**法的リスクがある**。ドキュメントとして明記しておく。

### リスク

- 試験問題・解説の**著作権は原則として試験実施団体等に帰属**する。過去問を無断で複製・翻案・配信すると著作権侵害になり得る。
- 「過去問道場」等の**Webサービスの掲載データ（独自解説・編集・DB構造）をコピーすると、そのサービスの著作権・利用規約違反**にもなり得る（参照元をそのまま写すのは NG）。

### 安全な進め方（優先順位）

1. **二次利用が明示許可された試験を最優先**
   - 電気技術者試験センター（電気工事士・電験）: 教育目的・**営利含め許諾/使用料不要**（出典明記が条件、改変時はその旨明記）。
   - 各**都道府県・省庁が公表する過去問**（登録販売者、一部国家資格）: 公表資料の利用条件を各団体ページで確認。
2. **公表問題があるものは出典を明記して利用**（改変時は改変明記）。利用条件は試験ごとに異なるため、リリース前に**実施団体の利用規約を必ず確認**する。
3. **公表されていない試験（例: 危険物・簿記ネット試験）は「オリジナル予想問題」として自作**する。既存過去問の丸写しは避け、出題範囲・論点から問題文・選択肢・解説を書き起こす（AIは下書き用途、最終確認は人手）。
4. **他社アプリ/Webの解説文をコピーしない**。解説は自作する。
5. リリース時に**免責・出典表記**をアプリ内に用意する。

> まとめ: 「① 公表・二次利用OKの試験を選ぶ → ② 出典明記 → ③ 非公表はオリジナル問題化」の順で、著作権リスクを回避しながら量産する。
