# ITパスポート アプリ ASO 確定版（リリース前）

対象ストア: App Store（日本）/ Google Play（日本）
方針: 日本国内・日本語のみ。基本無料＋一部課金。初期はインストール数最大化。

---

## 0. 重要原則（2026 最新仕様の要約）

- **キーワード欄(100字)は使い切る**。ユーザー非表示の索引枠なので空けると損。カンマのみ・スペースなし・末尾カンマなし。
- **タイトル/サブタイトルは詰め込まない**。索引＋コンバージョン両用の最重要表示領域。重要語を前方へ（途中で「…」省略表示されるため）。
- **iOS 日本ストアは「日本語＋英語(US)」の2ロケールを索引** → 英語(US)ロケール追加で実質キーワード枠が約200字に倍増。語はロケール間で重複させない。
- **競合アプリ名・商標はメタデータに入れない**（Apple 2.3.7 / 5.2 / 4.1、Google IP・なりすまし）。リジェクト・削除・アカウント停止・商標クレームのリスク。一般名詞（基本情報・情報処理など）はOK。
- **概要(説明文)は順位非関与だがCVRと会話型検索(Gemini/Ask Play)に重要** → 機能羅列でなく「課題解決の自然文＋FAQ」。
- **スクショ内テキストはOCR索引化(iOS)** → キャプションにキーワードを入れる。
- **インストール後の継続率(Day1/7/30)・セッション頻度が2026のランキング要因** → リテンション施策を初速で。

## 1. ブランド名（確定）

- **ブランド名：パスタディ**（Pass＝合格 ＋ Study＝学習）
- **英字ロゴ：PaStudy**（Pass+Study を視覚的に分割し意図を明快化）
- **タグライン（任意）：受かる勉強を、毎日。**
- **運用型：「試験名（一般語＝検索流入）＋ パスタディ（固有名＝指名・信頼）」**

### 採用理由
- **指名検索の堀が最大**：造語のため衝突ゼロ・商標化でき、自社が指名検索を独占できる。
- **意味の合致**：合格(Pass)＋学習(Study)で、ユーザーの優先度1位・2位を内包。
- **アンブレラ適性**：試験非依存。全資格アプリで流用可（横展開）。
- 検索流入は先頭の試験名（例「ITパスポート過去問」）が担うため、ブランドは育てる前提でよい（`スタディサプリ`／`abceed`／`mikan` も当初は無名の造語）。

### 横展開の命名規則（アンブレラ）
- 形式：`{試験名}{過去問/対策など} - パスタディ`
- 例：`ITパスポート過去問 - パスタディ` / `基本情報過去問 - パスタディ` / `FP3級 過去問 - パスタディ`
- ストア内でアプリ名は一意である必要があるが、先頭の試験名が異なるため重複しない。
- 商標は「パスタディ」を1件取得すれば全アプリをカバー。

### ブランド名 検証ログ（既存衝突調査・2026/06）
リリース前に **J-PlatPat の称呼類似検索（全角カタカナ）** で最終確認すること。Web調査で以下が判明：

| 候補 | 判定 | 理由 |
|---|---|---|
| **パスタディ** | 採用 | 学習アプリでの使用・近接社名ともに未検出（クリーン） |
| Iパス道場 | 除外 | 「Iパス」内包で横展開に不適（試験依存） |
| パストレ | 除外 | path tracing(3DCG)の定番略称と衝突／社名パストラーレと称呼近 |
| マナパス | 除外 | 文科省「マナパス（学びのパスポート）」が稼働中 |
| パスログ | 除外 | 認証「パスロジ／PassLogic」と称呼近・IT領域で紛らわしい |
| スタパス | 除外 | 「スタディパス」(受験)・「スタパ」(学習塾)・美容系スタパスと衝突 |
| スキパス | 除外 | すき家「Sukipass(スキパス)」が著名・公式アプリ内販売 |
| ミニツク | 除外 | フェリシモ通信講座「ミニツク」／Ruby学習サイトと衝突 |
| テッパン | 除外 | オンスク「TEPPAN」・FPキャンプ「TEPPEN」・スタサプ「TEPPAN英単語」と衝突 |

> 教訓：意味が明快な実在語は資格学習ドメインで先に埋まっている。造語（パスタディ）は定着を育てる必要があるが、独自性・所有可否で有利。

以下のメタデータはすべて **パスタディ採用** で確定。

## 10. アプリプレビュー動画（iOS / Google Play）

> 2025-2026のデータでCVR +20〜40%の効果が確認されている。App Store検索結果で自動再生されるため競合との差別化に直結。

### 仕様
- **尺**: 15〜20秒（最大30秒だが短いほど視聴完了率が高い）
- **無音再生前提**: テキストキャプション＋UIアニメーションで訴求
- **冒頭3秒で価値訴求**（ここで離脱されると逆効果）

### 構成案（既定ページ用・15秒）

| 秒数 | 画面 | テキストキャプション |
|:---:|:---|:---|
| 0-3 | 課題提示（暗い画面→アプリ起動） | 「忙しくて勉強できない…」 |
| 3-8 | 片手スワイプで問題を解く操作 | 「1回5分。片手でサクサク」 |
| 8-12 | 忘却曲線の自動復習アニメーション | 「忘れる前に自動復習」 |
| 12-15 | 合格予測スコア＋レーダーチャート | 「合格までの距離が見える」 |
| 15 | CTAカード | 「今すぐ無料で始めよう」 |

### CPP用差し替え版
- **CPP-A（独学/スキマ）用**: スワイプ操作・通勤電車シーンを強調
- **CPP-B（直前対策）用**: 模擬試験・スコア推移を強調

## 11. アプリアイコン最適化

> アイコンはストア閲覧時のCVRに直結する最重要ビジュアル。A/Bテストで最適化する。

### 設計方針
- **教育アプリで高CVRの特徴**: 明るいグラデーション、シンプルなシンボル、読みやすい1〜2文字
- ブランド「パスタディ」のロゴマーク「P」をベースに
- 背景: ブルー〜グリーンのグラデーション（信頼感・成長のイメージ）
- 小さいサイズ（検索結果一覧）でも判別できるシンプルさ

### A/Bテスト計画
- **iOS**: App Store Connect「プロダクトページ最適化」でアイコンテスト
- **Google Play**: ストア掲載エクスペリメントでアイコンテスト
- テスト候補: 2〜3バリエーション（グラデーション色・シンボル形状・文字有無）

## 12. プレローンチ戦略（初速48時間の最大化）

> Appleのアルゴリズムは新規アプリに一時的な可視性ブースト（ハネムーン期間）を与える。この48時間のDL数・評価数がその後のオーガニック順位を左右する。「審査通過→即公開」は初速ゼロでスタートするリスクがある。

### 事前予約（Pre-Order）の活用
- App Store Connectでリリース日を**2〜3週間後**に設定しPre-Orderを有効化
- Pre-Order中にストアページがインデックス開始 → 早期のキーワード順位獲得
- リリース日に事前予約者へ一斉ダウンロード → チャートブースト

### SNS・マーケティング

| 施策 | 詳細 | タイミング |
|:---|:---|:---|
| X(Twitter)公式アカウント開設 | 「パスタディ公式」でITパスポート受験者コミュニティに参加 | Pre-Order開始と同時 |
| カウントダウン投稿 | 7日前・3日前・前日・当日 | リリース日から逆算 |
| 簡易ランディングページ | メールアドレス収集 + App Storeリンク | Pre-Order開始と同時 |
| ITパスポート受験者向けQA | Yahoo知恵袋・Studyplus等で学習法の回答にアプリを自然紹介 | Pre-Order期間中 |

### TestFlightテスター活用
- 100〜500人規模のベータテスター募集（SNS・知人・学習コミュニティ）
- テスターにDay1でのレビュー投稿を依頼（強制ではなく自然な形で）
- フィードバックを反映し品質を最終確認
