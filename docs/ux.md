## 2. 画面別UX評価

評価軸: ★★★★★（コンセプト整合性 / 明瞭さ / インタラクション品質）

### HomeScreen（ホーム画面）

**役割**: ゲームタイプとレベルを選択させ、適切な学習へ誘導する

```
現状のUI構成:
  ┌─────────────────────────────────┐
  │  ⚙️（設定）                      │
  │                                 │
  │  [ロゴ + ゲームタイプ名]          │  ← PageView（左右スワイプ）
  │  ● ○（ページドット）             │
  │  BLOCK PUZZLE                   │
  │                                 │
  │  [英検5] [英検4] [英検3] [準2]  │
  │  [英検2] [600]  [700]  [800]   │  ← 4列グリッド
  │  [900]                          │
  │  （高得点 🏆・習熟度バー○○%）     │
  └─────────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★☆☆ | ゲームと学習の両立感は伝わるが、ゲームタイプの違いが視覚的に分かりにくい |
| 明瞭さ | ★★★★☆ | 習熟度バー（○○%）で学習状況が一目で分かる。レベル選択は直感的 |
| インタラクション | ★★★★☆ | PageView のスワイプは自然。カードのハイスコア表示も継続動機になる |

**具体的な問題点:**

1. **Block Puzzle と Number Merge の差別化**: PageView でゲームタイプを切り替えられるが、ロゴ下のラベルが変わるだけで「どう違うゲームか」が視覚的に伝わらない。初見ユーザーが違いを理解せずに進む可能性がある。

2. **英語学習アプリとしての第一印象**: 英検・TOEICのレベルカードが並んでいるため「学習アプリっぽい」第一印象になる。ゲームを楽しみたいユーザーへの訴求力が弱い可能性がある。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 中 | ゲームタイプ切り替え時に盤面プレビュー（小さなアニメーション）を表示 | 2ゲームの違いを直感的に伝える |
| 低 | レベルカードに習得語数（例：「230/680語」）を表示 | 学習進捗の具体化 |

### QuizStartSheet（クイズ開始ボトムシート）

**役割**: 学習フィルターと出題順序を選んでゲーム開始へ誘導する

```
現状のUI構成:
  ┌─────────────────────────────────┐
  │  ── (ドラッグハンドル)            │
  │  EIKEN 5                        │
  │                                 │
  │  習熟度フィルター（横スクロール）   │
  │  [全て:110] [New:80] [苦手:20]  │
  │  [学習中:8] [得意:2] [完璧:0]   │
  │                                 │
  │  出題順序                        │
  │  [おまかせ] [苦手優先] [得意優先]  │
  │                                 │
  │  [ START ]                      │
  └─────────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | 習熟度フィルターにより「苦手だけ集中的に」という学習意図を反映できる |
| 明瞭さ | ★★★☆☆ | 「おまかせ / 苦手優先 / 得意優先」の意味が直感的でない部分がある |
| インタラクション | ★★★★☆ | 横スクロールチップは操作しやすい。STARTボタンの無効状態（語数不足時）も適切 |

**具体的な問題点:**

1. **「おまかせ」の中身が不透明**: SRS・重み付きランダムであることが分からず、「適当に出す」と誤解されうる。
2. **出題順序の価値**: 「得意優先」がなぜ有用かが伝わらない（復習効率的・自信をつけるなど）。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 低 | 「おまかせ」に「（SRS 科学的復習）」などのサブラベルを追加 | ユーザーの信頼感向上 |

### BlockPuzzleScreen（ブロックパズル画面）

**役割**: ブロックパズルとクイズを交互に体験させ、双方向ループを感じさせる

```
ブロックフェーズ:
  ┌─────────────────────────────────┐
  │ スコア × 倍率                    │
  │ [8×8グリッド]                    │  ← ブロック配置
  │ [ピーストレイ: 英語/日本語ラベル]  │  ← 次のクイズ単語が表示
  └─────────────────────────────────┘

クイズフェーズ（3問）:
  ┌─────────────────────────────────┐
  │ [スワイプカード]   ← stage 0〜2  │
  │   apple + 🔊                    │
  │  ↑りんご ↓orange                │
  │                                 │
  │ [letterTapカード] ← stage 3     │
  │   日本語: りんご                  │
  │  a p _ _ e                      │
  │                                 │
  │ [タイピングカード] ← stage 4     │
  │   日本語: りんご                  │
  │  [ テキストフィールド ]           │
  └─────────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | ゲーム↔学習ループは体験できている |
| 明瞭さ | ★★★☆☆ | ステージ進行に伴う出題形式の変化が事前に説明されない |
| インタラクション | ★★★★★ | スワイプ操作・TTS・ヒントアニメーションは完成度高い |

**具体的な問題点:**

1. **クイズ形式の突然の変化**: stage 0〜2 はスワイプ4択なのに、stage 3 で突然 letterTap に変わる。ユーザーが「バグ？」と感じる可能性がある。
2. **ピース難易度変化の「見えない因果関係」**: 「正解するといいピースが来る」というコアループがプレイ中にユーザーへ伝わっていない可能性が高い。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | stage 3 到達時に「スペル入力モードに変わりました」の1回限りトースト | 混乱防止 |
| 中 | 初回プレイ時のみ「正解するといいピースが来る！」コーチマーク | コアループの理解促進 |

### NumberMergeScreen（ナンバーマージ画面）

**役割**: 2048系パズルと英単語クイズを融合させた新しいゲーム体験を提供する

```
マージフェーズ:
  ┌─────────────────────────────────┐
  │ スコア                           │
  │ [4×4グリッド（数字タイル）]        │  ← スワイプして合体
  │ [スポーンスロット]                 │  ← 次のタイルプレビュー
  └─────────────────────────────────┘

クイズフェーズ（3問）:
  ← BlockPuzzle と同じ QuizCard を使用
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★☆☆ | 2048系×英単語クイズの融合は新しいが、因果関係が分かりにくい |
| 明瞭さ | ★★★☆☆ | 「クイズ正解→どのタイルが有利になるか」が視覚的に不明瞭 |
| インタラクション | ★★★★☆ | タイルのスワイプ操作・合体アニメーションは直感的 |

**具体的な問題点:**

1. **クイズとの連動が不透明**: Block Puzzle は「正解→ピース品質向上」が比較的理解しやすいが、Number Merge は「クイズ正解→何が変わるか」が分かりにくい。
2. **Block Puzzle との使い分け**: 2ゲームが並んでいるが、「どちらを選ぶか」の判断基準をユーザーが持てない。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | ホーム画面のゲームタイプ切り替え部分に「Block: ライン消去 / Merge: 数字合体」の一言説明を追加 | ゲーム選択の迷い解消 |

```
  ┌─────────────────────────────────┐
  │ GAME OVER                       │

  │                                 │
  │ [ Play Again ]  [ Home ]        │
  └─────────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | 苦手単語リスト表示は学習継続動機に直結している |
| 明瞭さ | ★★★★☆ | 正解率・苦手単語は分かりやすい |
| インタラクション | ★★☆☆☆ | ゲームオーバー直後のインタースティシャル広告が体験を断絶する |

**具体的な問題点:**

1. **広告の割り込みタイミング**: ゲームオーバー時に即座にインタースティシャル広告が表示される。「苦手単語を確認しよう」という感情的な瞬間を広告が遮断する。学習モチベーションが高まっているタイミングでのUX破壊。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | 広告を「Play Again」ボタン押下後に表示（タイミング変更） | UX断絶を大幅緩和・プレミアム購入意欲も高まる |
| 中 | 「次回レビューできる語: N語」を表示 | D7 Retention向上の動機付け |

## 3. 観点別総合評価

### 3-1. 初回体験（オンボーディング）

**現状評価: ★★★☆☆**

```
インストール → 起動 → ホーム画面の現状フロー:

[App Store] → [インストール] → [ホーム（レベルカードグリッド）]
                                       ↓
                             どのレベルを選べばいいか分からない
                             Block Puzzle と Number Merge の違いが不明
```

**強み**:
- レベルカードに習熟度バー（○○%）があり、進捗が視覚的に分かる
- QuizStartSheet がシンプルで、難しい設定なしに START できる

**弱み**:
- 「どのレベルから始めるべきか」の案内がない
- ゲームタイプの違いの説明がない
- チュートリアルなし

---

### 3-2. 継続・習慣化

**現状評価: ★★☆☆☆**

```
習慣化フックの現状:

✅ SRS（復習タイミング管理）  ← 仕組みはある
✅ レベルカードの習熟度バー   ← 進捗の可視化あり
❌ プッシュ通知               ← 未実装
❌ ストリーク表示             ← なし
❌ 「今日N語レビューできます」 ← ホーム画面に表示なし
△  ゲームオーバー後の苦手語表示 ← あるが翌日へのフックが弱い
```

**本質的な課題**: SRSアルゴリズムは裏で機能しているが、**ユーザーに「今日また来る理由」が見えていない**。

---

### 3-3. ゲームと学習のバランス

**現状評価: ★★★★☆**

**強み**:
- ピース/タイル難易度が正解率に連動している点は独自の差別化
- TTS（音声）が学習の多感覚化に貢献
- SRSによる復習タイミング管理は科学的根拠あり
- stage 3 以降のletterTap/タイピング形式はより深い記憶定着に貢献

**弱み**:
- 因果関係が「暗黙的」（正解→良いピース/タイルの説明なし）
- ステージ進行で突然クイズ形式が変わる点が混乱を生む可能性

---

### 3-4. マネタイズ体験

**現状評価: ★★★☆☆**

**強み**:
- バナー広告は常時表示だが、ゲームのコアUXは妨げていない

**弱み**:
- ゲームオーバー時のインタースティシャルは学習体験を最も妨げるタイミング
- ペイウォールへの能動的な誘導がほぼない
- ペイウォールの価値訴求が「広告なし」のみで弱い

---

## 4. 総合スコアと優先アクション

| コンセプト | 実装状況 | ユーザーへの伝達 | ギャップ |
|-----------|---------|---------------|---------|
| ゲームが学習を引き出す | ✅ SRS実装済み | ❌ 毎日来る理由がユーザーに見えない | 大 |
| 学習がゲームを強化する | ✅ ピース/タイル難易度連動 | ❌ 因果関係が説明なし | 大 |

| 🔴 高 | ホーム画面に2ゲームの一言説明追加 | NMへの利用分散 | 低 |
