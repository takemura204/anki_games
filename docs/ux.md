## 2. 画面別UX評価

評価軸: ★★★★★（コンセプト整合性 / 明瞭さ / インタラクション品質）

---

### HomeScreen（ホーム画面）

**役割**: 4つのモードを提示し、適切なモードへ誘導する

```
現状のUI構成:
  ┌─────────────────────────────┐
  │  ⚙️                         │  ← 設定アイコン（右上）
  │  [テーマプレビューブロック]    │
  │   BLOCK PUZZLE              │  ← タイトル
  │   (サブタイトル)             │
  │                             │
  │ [QUEST] [CLASSIC] [TIME] [WORD] │  ← 4つのモードカード（横スクロール）
  │          (等幅・同デザイン)       │
  │                             │
  │ 英検5 英検4 英検3 TOEIC ...  │  ← Word Modeカード内レベルチップ
  └─────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★☆☆ | Word Modeが「核心機能」なのに見た目が他の3モードと同等 |
| 明瞭さ | ★★★☆☆ | バッジ表示が不統一（ENDLESS / BEST XX / LEVEL N）|
| インタラクション | ★★★★☆ | レベルチップタップで直接ハブへ飛べる点は良い |

**具体的な問題点:**

1. **Word Modeが埋もれている**: 4つのモードが横スクロールで等幅・同デザイン。アプリの核心である Word Mode が差別化されていない。初見ユーザーが「CLASSIC」から試す可能性が高い。

2. **バッジの不統一**: Classic Mode は未プレイ時に `"ENDLESS"` と表示されるが、Quest は `"LEVEL 1"` 、Word Mode は `"QUIZ × PUZZLE"` と説明的で異なる。ユーザーが何を意味するか迷う。

3. **レベルチップのUX**: タップすると確認なしに即座にハブ画面へ遷移する。チップを選んだ感触（アニメーション150ms）は短く、「本当に選べたのか」が分かりにくい可能性。

**改善提案（優先度順）:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | Word Modeカードをビジュアル的に強調（サイズ大・色・バッジ変更） | Word Mode利用率30%→50%への寄与 |
| 中 | バッジ表現の統一（未プレイ時は全モード説明文を同形式に） | 初回体験の混乱低減 |

### WordRangeSelectorScreen（学習ハブ画面）

**役割**: 学習状況を可視化し、適切な範囲を選択してSTARTへ誘導する

```
現状のUI構成:
  ┌─────────────────────────────┐
  │ ← [英検5]              ⋮   │  ← タイトル（選択レベル名）+ 単語一覧ボタン
  │                             │
  │ [ドーナツチャート | 内訳]    │  ← 習熟度ダッシュボード（ローディング挟む）
  │ [得意TOP5 | 苦手TOP5]       │  ← 2カラムランキング
  │                             │
  │ 学習範囲: [すべて][苦手+未]  │  ← 横3チップ
  │           [苦手のみ]         │
  │                             │
  │ ジャンル:  [頻出50選]        │  ← 無料
  │           [🔒自然][🔒日常]... │  ← プレミアムロック
  │                             │
  │       [ START (N語) ]        │
  └─────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | 学習状況の可視化はSRSの動機付けに直結している |
| 明瞭さ | ★★☆☆☆ | 初回ユーザーには情報量が多い。ドーナツグラフが「何を意味するか」が説明なし |
| インタラクション | ★★★★☆ | フィルターとSTARTの流れは直感的 |

**具体的な問題点:**

1. **初回ユーザーへの過剰情報**: 初めて画面を開くとDBが空のため、ドーナツグラフには「未学習○○語」しか表示されない。ランキングも空。→「これは何の画面？」という印象になりやすい。

2. **ローディング体験**: DBからデータ取得する間 `CircularProgressIndicator` が表示される。理由の説明がなく、「重いアプリ」と感じさせる可能性。

3. **ロックコンテンツの文脈**: ジャンルフィルターのロックアイコンが並ぶが、「なぜロックされているのか」の説明がない。初見でペイウォールの意味が伝わらないと、不完成なアプリに見える。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | 初回のみ「まずはSTARTを押してみよう」コーチマークを表示 | 初回離脱防止 |
| 中 | ローディング中にプレースホルダー（スケルトンUI）を表示 | 「重い」印象を軽減 |
| 低 | ロックジャンルに「プレミアムで解放」のツールチップを追加 | ペイウォール文脈の明確化 |

### BlockPuzzleScreen — クイズフェーズ（ゲームメイン画面）

**役割**: ブロックパズルとクイズを交互に体験させ、双方向ループを感じさせる

```
ブロックフェーズ:
  ┌─────────────────────────────┐
  │ スコア × 倍率               │
  │ [8×8グリッド]               │  ← ブロック配置
  │ [ピーストレイ: 英語/日本語]  │  ← 次のクイズ単語がピース下に表示
  └─────────────────────────────┘

クイズフェーズ（3問）:
  ┌─────────────────────────────┐
  │                             │
  │  [スワイプカード]            │  ← 英単語 + 🔊アイコン
  │    apple                   │
  │  ↑りんご ↓orange            │  ← 4方向に選択肢
  │  ←apricot →appule           │
  │                             │
  └─────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | ゲーム↔学習ループは体験できている |
| 明瞭さ | ★★★☆☆ | ピース難易度がなぜ変わるかの説明がユーザーに伝わっていない |
| インタラクション | ★★★★★ | スワイプ操作・TTS・ヒントアニメーションは完成度高い |

**具体的な問題点:**

1. **ピース難易度変化の「見えない因果関係」**: 「正解するといいピースが来る」というコアループが、プレイ中にユーザーへ伝わっていない可能性が高い。ピーストレイに単語は表示されているが、「なぜこのピースなのか」の説明がない。コアコンセプトの最重要な体験なのに暗黙的になっている。

2. **ブロックフェーズでのピーストレイ**: 次ラウンドのクイズ単語がピース下に表示されているが、その意味が初見では分からない（「英語・日本語が書いてあるな」程度）。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | 初回プレイ時のみ「正解するといいピースが来る！」ポップアップを表示 | コアループの理解促進、継続動機の強化 |
| 中 | ピース付与時に「Good piece! 🎉」のフィードバックをアニメーションで表示 | 学習とゲームの因果関係を体感させる |

### GameOverOverlay（ゲームオーバー画面）

**役割**: セッション成果をまとめ、次回プレイへの橋渡しをする

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★★☆ | 苦手単語リスト表示は学習継続動機に直結している |
| 明瞭さ | ★★★★☆ | 正解率バー・苦手単語は分かりやすい |
| インタラクション | ★★☆☆☆ | ゲームオーバー直後のインタースティシャル広告が体験を断絶する |

**具体的な問題点:**

1. **広告の割り込みタイミング**: ゲームオーバー時、振動・サウンドと同時にインタースティシャル広告が表示される。「苦手単語を確認しよう」という感情的な瞬間を広告が遮断する。特に学習モチベーションが高まっているタイミングでのUX破壊。

2. **「次へ」の動線の弱さ**: ゲームオーバーで苦手単語を見た後、「ではもう一度復習しよう」という自然な行動フローにつながっているか要確認。（Play Again / 単語確認 / ホームの選択肢が明確か）

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 高 | 広告をゲームオーバー画面の「Play Again」ボタン押下後に表示（タイミング変更） | UX断絶を大幅緩和。プレミアム購入意欲も高まる |
| 中 | 苦手単語タップ → その場でTTSで発音再生 | 「次はこれを覚えよう」という動機を強化 |

### PaywallBottomSheet（プレミアム購入画面）

**役割**: プレミアム価値を伝え、購入へ転換する

```
現状のUI構成:
  ┌─────────────────────────────┐
  │ ── (ドラッグハンドル)         │
  │  🔓 Block. Premium          │
  │                             │
  │  ✓ 広告を完全に非表示        │
  │  ✓ 全ジャンルの単語を解放    │
  │                             │
  │  [ ¥480/月で始める ]        │
  │  [ 購入を復元する ]          │
  │                             │
  │  利用規約   プライバシーポリシー│
  └─────────────────────────────┘
```

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| コンセプト整合性 | ★★★☆☆ | 「広告除去」は動機になるが「全ジャンル解放」の価値訴求が弱い |
| 明瞭さ | ★★★★☆ | シンプルで迷わない。利用規約リンクも完備 |
| インタラクション | ★★★★☆ | ローディング・エラー・復元フィードバックが適切 |

**具体的な問題点:**

1. **価値訴求の薄さ**: 機能リストが2行のみ。「なぜ¥480を払う価値があるのか」のストーリーがない。例えば「英検3級の単語を全て網羅できる」「広告なしで集中して学習できる」という具体的なベネフィットが伝わっていない。

2. **「購入を復元する」ボタンの文脈**: 初見ユーザーには「復元？何を？」と意味が分からない。Apple/Googleの審査要件で必要なボタンだが、説明テキストがあると良い。

3. **接触タイミングの問題**: ロックされたジャンルカードをタップした時のみ表示される。より能動的な誘導（例：Word Mode 10回プレイ後のスマートタイミング）がない。

**改善提案:**

| 優先度 | 提案 | 期待効果 |
|--------|------|---------|
| 中 | 機能リストに学習価値の具体性を追加（「英検5〜2級 全500語のジャンル別学習」など） | CVRの改善 |
| 低 | 「復元する」ボタンの下に「以前ご購入の場合」という補足テキストを追加 | ユーザー混乱の低減 |

## 3. 観点別総合評価

### 3-1. 初回体験（オンボーディング）

**現状評価: ★★☆☆☆**

```
インストール → 起動 → ホーム画面の現状フロー:

[App Store] → [インストール] → [ホーム画面4モード]
                                       ↓
                             説明なし・チュートリアルなし
                             どれを押せばいいか不明
```

**最大の問題**: アプリの核心である「ゲームで学ぶ双方向ループ」が、プレイ開始前にユーザーへ一切説明されない。コンセプトが伝わる前にパズルを始めてしまうと、Word Modeを選ばないまま離脱するリスクがある。

**比較（競合）**:
- Duolingo: 起動時に「どの言語を学びたい？」「1日何分できる？」の設定で即座にパーソナライズ
- Drops: 最初の3語を無料体験させてから登録画面
- Block.: 説明なし・全モード同列提示

**改善優先度: 高**

| アクション | 難易度 | 効果 |
|---------|--------|------|
| 初回起動時にWord Modeを推奨するコーチマーク（1〜2枚） | 低 | D1 Retention改善 |
| Word Modeカードの視覚的強調 | 低〜中 | Word Mode利用率改善 |
| 初回プレイ時の「正解→いいピース」の説明 | 低 | コアループ理解促進 |

### 3-2. 継続・習慣化

**現状評価: ★★☆☆☆**

```
習慣化フックの現状:

✅ SRS（復習タイミング管理）  ← 仕組みはある
❌ プッシュ通知                ← 未実装
❌ ストリーク表示              ← なし
❌ 「今日N語レビューできます」  ← ホーム画面に表示なし
△  ゲームオーバー後の苦手語表示 ← あるが翌日へのフックが弱い
```

**本質的な課題**: SRSアルゴリズム（Phase 10完了）は裏で機能しているが、**ユーザーに「今日また来る理由」が見えていない**。Duolingoが強力なのはSRSより「ストリーク」という可視化された習慣のフックがあるから。

**改善優先度: 高（D7 Retention最重要レバー）**

| アクション | 難易度 | 対応KPI |
|---------|--------|---------|
| プッシュ通知（SRS復習リマインダー） | 中 | D7 Retention |
| ホーム画面に「今日のレビュー語数」バッジ表示 | 低 | DAU/MAU |
| ゲームオーバー後に「明日レビューできる語: N語」を表示 | 低 | D7 Retention |

### 3-3. ゲームと学習のバランス

**現状評価: ★★★★☆**

```
バランスの評価:

                 [学習が強い]
                      |
    Duolingo ─────────|
                      |
    Block. ───────────+─────── [ゲームが強い]
    (理想位置)         |            ↑
                      |         1010!
                 [ゲームが強い]
```

Block.は「ゲームと学習の中間」というポジションをほぼ達成できている。ただし以下の点で改善余地がある：

**強み:**
- ピース難易度が正解率に連動している点は独自の差別化
- TTS（音声）が学習の多感覚化に貢献
- SRSによる復習タイミング管理は科学的根拠あり

**弱み:**
- 因果関係が「暗黙的」（正解→良いピースの説明なし）
- パズルを楽しみたいユーザーには3問クイズが「邪魔」に感じる可能性
- クイズフェーズとブロックフェーズの比率が最適化されているか未検証

**改善提案:**
- 「ゲームに集中したい日」用に クイズなしモード（Classic）が既にあるのは正解。ただしその使い分けをユーザーに伝える必要がある。

### 3-4. マネタイズ体験

**現状評価: ★★★☆☆**

```
マネタイズ体験の現状マップ:

無料ユーザーの体験:
  ┌──────────────────────────────┐
  │ [バナー広告が画面下に常時]    │  ← 学習中ずっと表示
  │                              │
  │ ゲームオーバー               │
  │    ↓ インタースティシャル広告 │  ← 学習成果確認の直前
  │    ↓ 苦手単語リスト          │
  └──────────────────────────────┘

プレミアム誘導:
  ジャンルカードタップ → ペイウォール（タイミングは正しい）
  ただし価値訴求が2行のみ
```

**強み:**
- バナー広告は常時表示だが、ゲームのコアUXは妨げていない
- ジャンルフィルターのロックは「使ってみたいのに使えない」という自然な動機づけになっている

**弱み:**
- ゲームオーバー時のインタースティシャルは学習体験を最も妨げるタイミング（前述）
- ペイウォールへの能動的な誘導がジャンルタップ以外にない
- 「広告なしで集中して学習したい」というニーズをペイウォールで伝えきれていない

**改善提案:**

| タイミング | 現状 | 改善案 |
