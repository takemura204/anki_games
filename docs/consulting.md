## 1. インストールされるか ── 4 / 10

### 評価理由

**強み（+側）:**
- コンセプトの独自性は本物。「SRS付きブロックパズル」は競合が存在しない
- ターゲットが2層（英語学習者 + パズルゲーマー）あり、検索ワードの幅が広い
- テーマ・スキン・エフェクトの視覚的完成度は高く、スクショ映えする

**リスク（−側）:**
- **アプリ名「Block.」が英語学習を一切想起させない。** ストアで英語学習を探しているユーザーには引っかからない。
- **ASOのみ = 実質ゼロから発見される必要がある。** Duolingo・Drops・Anki が上位を占める「英語学習」カテゴリに割り込むことは、有料UAなしでは極めて困難。
- **カテゴリ設定の矛盾。** パズルゲームとして出せば英語学習者に届かず、教育アプリとして出せばゲーマーに届かない。両者を引き寄せるキーワード設計が必要だが、難易度が高い。
- **ユーザーレビューゼロでのスタート。** 社会的証明がない状態では転換率3%の達成は楽観的。
- **ユーザーテスト未実施。** 現在の設計はすべて仮説。「スワイプクイズが楽しい」「ピース連動が面白い」が本当にそうかは未検証。

### 現実的な見通し

```
想定シナリオ（ASOのみ）:
  ストアインプレッション → 転換率2〜3% → 月間DL: 50〜200
  6ヶ月累計: 300〜1,200 DL（KPI合格ライン1,000 DLはギリギリ〜下回る）

SNS（X/TikTok）を加えた場合:
  バズ1本で +500〜2,000 DLの可能性あり
  → 集客チャネルの拡張が最大のレバー
```

### 優先アクション

| 優先度 | アクション | 難易度 |
|--------|---------|--------|
| 🔴 最高 | **アプリのサブタイトルに「英単語×ブロックパズル」を明示**（App Store Subtitle 30文字枠） | 低 |
| 🔴 最高 | **スクリーンショット1枚目をWord Modeの瞬間に変更**（クイズ回答→ピース付与の連続シーン） | 低 |
| 🔴 高 | **開発ログ・コンセプト動画をX/SNSで発信**（個人開発者のストーリーはバズりやすい） | 中 |
| 🟡 中 | **TestFlightで10〜20人に試してもらいレビューの種を作る** | 低 |

## 2. 継続されるか ── 5 / 10

### 評価理由

**強み（+側）:**
- SRSアルゴリズムは科学的根拠があり、正しく機能すれば「今日来る理由」を自動生成できる
- ゲームの「もう1回」中毒性はブロックパズルとして十分に機能している
- TTS音声は多感覚学習として効果的。競合の Drops と同水準

**リスク（−側）:**
- **SRSが動いているが、ユーザーに見えていない。** 「今日N語レビューできます」という表示がホーム画面にない。仕組みが存在しても、通知されなければD7で離脱する。
- **プッシュ通知なし = 最大の習慣化ボトルネック。** Duolingo の継続率はSRSではなくストリーク通知で作られている。仕組みだけあっても動機づけがなければ人は戻ってこない。
- **初回体験でコアコンセプトが伝わらない。** 「正解すると良いピースが来る」という双方向ループを知らずにプレイを終えるユーザーが多数出る。

### D7 Retention 試算

```
現状（何もしない場合）: 推定 5〜7%  ← KPI合格ライン12%を大きく下回る
プッシュ通知実装後:    推定 10〜14% ← 合格ライン付近
+ ストリーク表示:      推定 13〜18% ← ストレッチ目標到達の可能性
```

### 優先アクション

| 優先度 | アクション | 難易度 | 期待D7改善 |
|--------|---------|--------|-----------|
| 🔴 最高 | **プッシュ通知（SRS復習リマインダー）** | 中 | +5〜8pt |
| 🔴 高 | **ホーム画面に「今日のレビュー語: N語」バッジ** | 低 | +2〜3pt |
| 🔴 高 | **初回起動時のコアループ説明**（「正解→良いピース」を1画面で） | 低 | D1改善 |
| 🟡 中 | **ストリーク表示**（連続学習日数） | 中 | +2〜4pt |

## 3. 課金されるか ── 4 / 10

### 評価理由

**強み（+側）:**
- ¥480/月はDuolingo（¥1,533/月）の1/3。価格障壁は低い
- インタースティシャル広告の不快さは、プレミアムへの「自然な動機」になりうる
- ジャンルフィルターのロックは「使いたいのに使えない」という設計として正しい

**リスク（−側）:**
- **無料の「頻出50選」が充実しすぎている。** 英検5級100語のうち50語が無料で学べるなら、多くのユーザーはそれで満足する。「もっと欲しい」と思わせるには、無料範囲が広すぎる可能性がある。
- **ペイウォールの価値訴求が薄い。** 「広告除去」+「全ジャンル解放」の2行のみ。「英検3級 全150語を完全網羅」「集中学習モード」など、具体的なベネフィットが言語化されていない。
- **課金までの自然な動線が1つしかない。** ジャンルカードのタップのみ。Word Mode を10回プレイしても、プレミアムへの誘導は一切発生しない。
- **Retention低下が課金率に直撃する。** MAUが小さければ、CVR2%でも絶対数が少なすぎてMRRにならない。

### CVR 現実試算

```
シナリオ A（何もしない）:
  6ヶ月MAU: 500 / CVR: 1% → 契約5件 → MRR ¥2,400

シナリオ B（優先アクション実施後）:
  6ヶ月MAU: 700 / CVR: 2% → 契約14件 → MRR ¥6,720

KPI合格ライン（¥10,000 MRR）に届くには:
  MAU 1,000 × CVR 2%  = 20件 × ¥480 = ¥9,600 ≒ 合格
  → MAUを増やすことがCVR改善より先決
```

### 優先アクション

| 優先度 | アクション | 難易度 |
|--------|---------|--------|
| 🔴 高 | **ゲームオーバー広告をPlay Again後に移動**（インタースティシャルの不快さをプレミアム動機に転換） | 低 |
| 🟡 中 | **ペイウォールに具体的ベネフィットを追加**（「英検5〜2級 全500語のジャンル別学習」等） | 低 |
| 🟡 中 | **Word Mode 10回プレイ後に1度だけプレミアム紹介を表示** | 中 |
| 🟡 中 | **無料範囲の見直し**（頻出50選を30語に絞るなど、もっと欲しいと思わせる設計に） | 低〜中 |

## 4. 総合評価とコンサルコメント

### このアプリの本質的なポジション

```
                高い独自性
                    ↑
              Block. (現在地)
                    |
低い発見可能性 ──────┼────── 高い発見可能性
                    |
                    ↓
                低い独自性
```

**Block. は「独自性は高いが発見されにくい」象限にいる。**

コンセプトの完成度は個人開発として非常に高い。SRS×パズルの融合は学術的にも正しい設計で、Duolingo・Drops・1010! のどの競合も持っていない体験を提供できる。

しかし現状では**「良い製品が森の中に隠れている」状態**。ユーザーが発見できない、発見しても継続動機が弱い、継続しても課金する理由を感じにくい、という3つのボトルネックが連鎖している。

### 最重要メッセージ

> **継続率（D7 Retention）の改善を最優先にすること。**

理由: インストール数・課金数はいずれも「継続するユーザー数」に依存する。MAUが小さければ課金の絶対数も小さい。MAUを増やすには広告費が必要だが、Retentionが低ければ広告費をかけても水が溜まらないバケツになる。

**リリース前に最低限やること（3つだけ）:**

1. **プッシュ通知を実装する** → D7 Retention が最大 +8pt 改善
2. **ストアのサブタイトルとスクショ1枚目を Word Mode に振る** → 転換率改善
3. **広告タイミングをPlay Again後に移動する** → 1〜2行の変更で体験を大幅改善

この3つだけで総合スコアは **4.3 → 6.0** 程度まで改善できると試算する。
